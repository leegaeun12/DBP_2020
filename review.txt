2. 새로 배운 내용
-보안 문제가 생기는 이유와 이를 방지하기 위한 대책 
: SQL 인젝션 문제 -> mysqli_real_escape_string()을 통해 인자로 들어온 데이터 중에서 SQL을 주입하는 공격에 사용될 수 있는 기호를 문자로 바꿔줌.
-UPDATE
:DB에 저장한 데이터를 수정하는 기능을 직접 구현해봄
-DELETE
:DB에 저장한 데이터를 삭제하는 기능을 직접 구현해봄

3. 문제가 발생하거나 고민한 내용 + 해결 과정
-atom에 코드 자동 정렬 패키지를 설치할 때 계속하여 설치가 불가능하다는 오류가 떴다. 윈도우 설정에서 기본앱 설정을 바꿔주어 해결하였다.
-update 쿼리를 복사 붙여넣기 하는 과정에서 테이블이 EPL 아닌 다른 테이블로 잘못 입력되어 오류가 생겼다. 다시 꼼꼼히 확인하여 EPL 테이블로 바꾸니 제대로 동작하였다.

4. 회고
+ : update와 delete를 php를 통해 직접 구현해 볼 수 있어 신기했다. 

- : 쿼리들의 의미와 코딩의 순차적 절차를 정확히 안다기보다는 단지 교수님의 강의를 따라 똑같이 코딩하였다. 이 점을 보완하기 위해 더 많은 공부를 해야겠다고 생각했으며 쿼리들의 의미를 정확히 파악하고 싶다.
